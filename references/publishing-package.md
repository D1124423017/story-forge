# Publishing Package Mode

Use this reference when the user wants to prepare a manuscript for submission, serialization, pitching, platform posting, series packaging, back-cover copy, query letters, synopsis, chapter titles, or commercial positioning.

## Core Principle

A publishing package should sell the story's promise without lying about the manuscript.

Do not invent a market position that the actual story cannot deliver. If the premise, protagonist, stakes, or ending are unclear, diagnose that first.

## Package Types

### Query Letter

Use for agent or publisher outreach.

Include:

* title, genre, word count, audience
* hook paragraph: protagonist, desire, conflict, stakes
* escalation paragraph: what gets harder and why the choice matters
* closing note: comps if user provides them, series status, author bio if available

Avoid:

* explaining the entire world
* vague praise words
* spoilers unless required
* imitating famous franchise wording

### Short Synopsis

Use for submissions or internal planning.

Include:

* beginning pressure
* first turn
* midpoint reveal or reversal
* low point
* climax and ending

Unlike back-cover copy, a synopsis may reveal the ending when the target requires it.

### Back-Cover / Platform Blurb

Use for readers.

Include:

* hook image or situation
* protagonist desire
* central obstacle
* stakes
* tonal promise

Withhold the final answer. Sell the question.

### Serialization Package

Use for web serials, platform fiction, newsletters, or episodic releases.

Plan:

* update cadence
* chapter length target
* episode hook pattern
* recap strategy
* cliffhanger discipline
* comment/reader expectation management
* season or volume break

### Chapter Title Strategy

Titles can signal:

* object
* vow
* lie
* location
* countdown
* character name
* question
* reversal
* symbolic image

Avoid titles that spoil the twist unless the genre expects dramatic irony.
