# Example 03: Writer's Block Diagnosis

Prompt:

```text
用 $story-forge 診斷我為什麼卡稿：主角已經知道反派是誰，也拿到神器，朋友也原諒他了，但我還有十章才到結局。
```

Expected high-standard output:

```text
【卡稿診斷】

【真正卡住的原因】
你太早花完了三個主要壓力源：謎團答案、力量工具、關係修復。讀者現在沒有下一個必須翻頁的問題。

【不是問題的部分】
不是你不會寫下一章，也不是設定不夠多。問題是現有結果太像結局前一章。

【立刻可用的修法】
把「拿到神器」改成 false victory：神器能用，但每次使用會刪掉一段和朋友有關的記憶。

【下一章應該完成的功能】
讓主角第一次使用神器，救下一個人，同時忘記朋友曾經為他承擔背叛。

【三個可選下一步】
1. 反派身分是真，但反派只是更大系統的代理人。
2. 朋友原諒是真，但原諒本身有條件與代價。
3. 神器是真，但它讓主角越接近勝利越不像自己。
```
